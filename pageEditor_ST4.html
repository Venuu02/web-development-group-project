Challenges and Lessons Learned

Challenges Encountered

Responsive Layout Across Multiple Pages

Ensuring a consistent responsive layout across pages like user_profile.html, sitemap.html, and content_ST4.html proved challenging. Each page had different structures (SVG sitemap, profile cards, content grids), which required refining shared styles and media queries in CSS to maintain a unified design across screen sizes.

Interactive Footer Visibility

The scroll-based animation for the footer initially did not trigger consistently on all pages. Fixing this involved adding scroll event listeners and checking viewport height vs. document height to toggle the .visible class accurately on scroll. This was applied consistently across all pages.

Image Scaling and Alignment

Pages like user_profile.html and content_ST4.html include high-resolution images and icons. On smaller devices, they either overflowed or became misaligned. The fix involved applying max-width, auto margin, and display: block styles for proper centering and scaling.

Semantic Navigation and Internal Linking

In the sitemap.html page, linking clickable SVG areas to different sections/pages was initially difficult due to SVG coordinate limitations. This was resolved using <a xlink:href> and well-positioned elements within the SVG canvas.

Lessons Learned

Consistent Class Naming and Layout Patterns

Using consistent class naming (like .section, .card, and .footer) made it easier to reuse layout and animation styles across different HTML files while maintaining design integrity.

Testing Across Resolutions

Checking layouts on desktop, tablet, and mobile revealed hidden issues like image overflow and misaligned buttons. Applying flexbox, grid, and responsive media queries helped solve these.

Progressive Enhancement with JavaScript

JavaScript was used minimally but effectively — for example, scroll-based footer animation and section fade-ins. These enhanced user experience without compromising accessibility or load speed. Proper scroll thresholds and DOM readiness checks were essential to avoid glitches.
